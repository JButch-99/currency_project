# _Currency Conversion_

#### By: _**JButch-99**_

#### _This project is a single API project. You are able to use this to determine how much USD is valued in another country's currency._

## Technologies Used

* HTML

* CSS

* JavaScript

* esLintRc

* babelRC

## Description
_This is a project made to practice writing code for making API projects. This project lets you find a country's currency value, compared to USD. Inputting a value in USD, you can see how much USD that currency is comparable or equal to. You are able to reset the prompt, and enter new values to reuse the project._
 
## Set-Up / Install Guide 

NOTE: If you do not see this, please click on [this link](https://github.com/JButch-99/currency_project).

1. _To begin, please click on the green button labeled "<> Code", located in the upper right-hand corner of your browser. Please click from the dropdown menu after clicking the green button, the "download ZIP" option. Alternatively, if you are using Git: You also have the option to_ "git clone https://github.com/JButch-99/currency_project" _instead._

2. _Once downloaded, you will need to extract the files to gain access to the project. Click "extract all" in the folder. If on Mac, double clicking it_

3. _Open the project inside an code editor of choice, using an terminal and typing "code." in it. Then, using `$ npm install` to download all packages._

4. _To create an .env file to allow for functionality: Please type in `touch .env` to create an env file. Please note due to .gitignore, this will not be saved to any repositories made._

5. _Inside your newly created .env file, please put `API_KEY=`, followed by your API Key from [the ExchangeRate-Api Website](https://www.exchangerate-api.com/)._

6. _Type in `$ npm run build` in your console to build this project in webpack._

7. _To create an development server, type in `$ npm run start` in your console to access the project for it's functionality._

8. _To check for any issues, please use `$ npm run lint` in your console._

9. _To ensure all tests are passing with complete coverage, please run `$ npm run test` in your console._
